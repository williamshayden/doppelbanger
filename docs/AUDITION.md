# Ableton Audition Gate

Use this checklist after the automated fast benchmark passes for AlbumDB songs 01, 04, and 10. Keep the generated benchmark JSON and this completed review together as release evidence.

## Session Setup

1. Import each target, doppelbanger output, and reference master onto separate aligned tracks.
2. Set Warp off for every file and confirm each clip starts at the same sample position.
3. Route all tracks through the same neutral monitoring path with master-bus processing disabled.
4. Loudness-match target, output, and reference for judgment. Do not treat a louder file as automatically better.
5. Compare full tracks plus representative low-end, dense, transient-heavy, and sparse sections.

A severe artifact includes clipping, unstable filter ringing, clicks, non-finite dropout, obvious image collapse, or a level jump not explained by the plan.

## Acceptance

- No fast-suite output has a severe artifact.
- At least two of three outputs are judged closer to their reference than the reconstructed target.
- Any `worse` rating includes a timestamp and a short reproducible description.
- The matching benchmark report passes true-peak, format, duration, memory, and realtime gates.
